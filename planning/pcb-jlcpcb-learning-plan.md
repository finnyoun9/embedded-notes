## Step 1：NE555 双 LED 交替闪烁器（流程验证）

> **目的：** 用最简单电路走通「原理图 → PCB → Gerber → 嘉立创下单 → 收到实物」全流程
> **设计时间：** 1.5 小时 | **到手：** 3 天 | **费用：** ~$2-8

### 电路拓扑

```
         VCC (5V)
           │
    ┌──────┴──────┐
    │    R1 1k    │          NE555 经典非稳态多谐振荡器
    │     │       │          频率 ≈ 1.44 / ((R1+2R2)·C1)
    ├─────┤       │          R1=1k, R2=100k, C1=10µF → f≈0.7Hz
    │  7  DIS     │
    ├─────┤       │          输出方波驱动两颗 LED：
    │    R2 100k  │          OUTPUT HIGH → LED1 亮, LED2 灭 (通过 Q1 反相)
    ├─────┤  NE555│          OUTPUT LOW  → LED1 灭, LED2 亮
    │  2  THR     │
    │  6  TRG     │
    ├──┬──┤  3 OUT├──┬───────────── LED1 ──[R3 220Ω]── GND
    │  │  └───────┘  │
    │ [C1 10µF]      ├── Q1(2N2222) Base ──[R4 10k]── GND
    │  │             │       │
    │ GND            │    Collector
    │                │       │
    │                │    LED2 ──[R5 220Ω]── VCC
    │                │
    └────────────────┴───── GND

    Pin 4 (RST) → VCC     Pin 8 (VCC) → 5V
    Pin 5 (CTRL) → C2 10nF → GND（去耦，防噪声误触发）
```

### 理论要点

**振荡频率推导：**
- 充电路径：VCC → R1 → R2 → C1 → GND，时间常数 τ_charge = (R1+R2)·C1
- 放电路径：C1 → R2 → DIS(7) → GND，时间常数 τ_discharge = R2·C1
- 555 内部比较器阈值：充电到 2/3 VCC 翻转，放电到 1/3 VCC 翻转
- t_high = ln(2)·(R1+R2)·C1 ≈ 0.693 × 101k × 10µF ≈ 0.70s
- t_low = ln(2)·R2·C1 ≈ 0.693 × 100k × 10µF ≈ 0.69s
- f = 1.44 / ((R1+2R2)·C1) ≈ 0.72 Hz — 肉眼可见交替闪烁

**为什么用 NPN 三极管反相：**
- 555 OUTPUT 直接驱动 LED1（高电平亮）
- Q1 (2N2222) 作共射开关：OUTPUT HIGH → Q1 饱和导通 → 集电极拉低 → LED2 灭
- OUTPUT LOW → Q1 截止 → 集电极被 R5 上拉到 VCC → LED2 亮
- 这样两颗 LED 天然交替，无需额外逻辑

### PCB 布局要点

| 要点 | 说明 |
|------|------|
| **C1 (10µF 电解) 靠近 555 Pin 2/6** | 定时电容走线尽量短，减少寄生电容影响频率 |
| **C2 (10nF 去耦) 紧贴 Pin 5** | 从 Pin 5 到 GND 走线 < 5mm |
| **电源入口加 100nF 去耦** | VCC 入口处并联 100nF 瓷片电容，滤除电源噪声 |
| **LED 限流电阻靠近 LED** | R3/R5 紧贴 LED 阳极 |
| **板框 30×30mm 足够** | 8 个元件 + 2 个 LED，双面板绰绰有余 |
| **Edge.Cuts 画完整闭合矩形** | 最容易忘的坑：板框不闭合 → 拒单 |
| **测试点引出** | GND、VCC、OUT(Pin3) 各加一个 φ1.5mm 测试焊盘 |

### 验证方法

收到板子后：
1. DP100 供电 5V，限流 100mA
2. 上电 → 两颗 LED 应该交替闪烁，周期约 1.4 秒
3. 逻辑分析仪接 Pin 3 (OUT)，验证方波频率 ≈ 0.7Hz，占空比 ≈ 50%
4. 如果频率不对：检查 R1/R2 焊错没有，C1 容值对不对

### 关键子电路 —— 为什么这样设计

#### 1. 电源：AMS1117-3.3 LDO

```
USB 5V ──→ AMS1117-3.3 ──→ 3.3V 系统电源
              │    │
         10µF+100nF  10µF (输出端)
              │
             GND
```

**为什么输入输出都要 10µF + 100nF 并联：**
- 10µF 电解/钽电容 — 低频滤波，应对 USB 供电的纹波（100-120Hz 整流纹波 + 负载跳变）
- 100nF 瓷片电容 — 高频去耦，应对数字电路产生的 MHz 级开关噪声
- 电解电容的 ESL（等效串联电感）导致高频阻抗上升，瓷片电容补上这个缺口
- **Layout 铁律：两个电容都尽量靠近 AMS1117 引脚，走线从 USB 先经过电容再到 LDO 输入，不能先到 LDO 再旁路**

#### 2. 每个 VDD 引脚的 100nF 去耦

```
STM32 有 4 个 VDD 引脚（23/35/47/7），每个都要在 2mm 内放一颗 100nF
```

**为什么每个都要：**
- 这是**面试高频问题**。答案：CMOS 逻辑门在每次翻转时从电源抽取一个瞬时电流尖峰（几个 ns 内几百 mA）
- 如果 4 个引脚共享一颗电容，走线电感会让尖峰时刻的引脚电压跌落，导致逻辑错误
- 每引脚独立去耦 → 电流环路最小化 → 电源完整性
- 布局时电容和 MCU 引脚**同层放置**，不经过过孔，环路面积最小

#### 3. 时钟：8MHz 晶振 + 22pF 负载电容

```
OSC_IN ──┬── 8MHz ──┬── OSC_OUT
         [22pF]     [22pF]
          │          │
         GND        GND
```

**负载电容计算（面试会问）：**
- 晶振 datasheet 标称 CL（负载电容），典型 12-20pF
- CL = (C1·C2)/(C1+C2) + C_stray
- C_stray ≈ 3-5pF（PCB 走线 + 引脚寄生电容）
- 如果 CL=18pF：需要 (C1·C2)/(C1+C2) ≈ 13-15pF → C1=C2≈27-30pF
- 如果用 22pF：(22·22)/(22+22) = 11pF，加 3-5pF stray ≈ 14-16pF，匹配 CL≈15pF 的晶振
- **Layout：晶振和电容紧贴 MCU，下方不走信号线，包地隔离**

#### 4. 复位电路：RC + 按键

```
3.3V ── R 10kΩ ──┬── NRST (Pin 4)
                  │
              [按钮] ── GND
                  │
              100nF ── GND (硬件去抖)
```

**NRST 引脚内部已有上拉（~40kΩ），为什么还要外加 10kΩ：**
- 内部上拉较弱，外部强上拉抗干扰更好（电磁噪声不会误触发复位）
- 10kΩ 上拉电流仅 0.33mA，不影响功耗
- 100nF 电容提供硬件去抖 — 按钮按下时的机械抖动（几个 ms）被电容吸收，NRST 不会看到多个复位脉冲

#### 5. BOOT 配置

```
BOOT0 ── 10kΩ ── GND     ← 默认 Flash 启动
       └── [排针跳线] ── 可切换到 3.3V ← 系统存储器启动（串口下载）

BOOT1 ── 10kΩ ── GND     ← 固定拉低
```

| BOOT0 | BOOT1 | 启动模式 |
|-------|-------|----------|
| 0 | x | **Flash 启动**（正常运行） |
| 1 | 0 | 系统存储器（内置 Bootloader，串口下载） |
| 1 | 1 | SRAM 启动（调试用） |

留跳线帽位置：正常工作时 BOOT0 接 GND，需要串口烧录时跳到 3.3V。

### PCB 布局约束

| 规则 | 数值 | 原因 |
|------|------|------|
| 板框 | ~50×50mm 或 80×50mm | 2 层板，JLCPCB 此尺寸 $2/5pcs |
| 电源走线宽度 | ≥ 0.5mm (20mil) | 承载 USB 5V / 3.3V 电流 |
| 信号走线宽度 | ≥ 0.25mm (10mil) | STM32 LQFP-48 引脚间距 0.5mm，能走出来 |
| 过孔 | 0.8mm pad / 0.4mm hole | 嘉立创推荐最小规格 |
| 晶振到 MCU | < 15mm | 越短越好，下方铺 GND 铜 |
| 去耦电容到引脚 | < 2mm | 环路最小化，优先于整齐美观 |

收到板子后：
1. 万用表测 3.3V 对 GND 不短路，5V 对 GND 不短路

### 关键设计决策

#### 1. 为什么加独立模拟 LDO（LP2985）

STM32 的 VDDA 给 ADC 和 DAC 供电。AMS1117-3.3 的输出噪声约 50-100µV RMS，对于 12 位 ADC（3.3V / 4096 ≈ 0.8mV/LSB）来说尚可，但如果数字部分（CH340、电机驱动）的地弹耦合进来，ADC 读数会抖动。

**方案：** LP2985-3.3 给 VDDA 单独供电。LP2985 噪声 ~30µV RMS，且在 100kHz 处 PSRR > 45dB，能有效隔离数字噪声。

**Layout 要点：** 模拟地 (AGND) 和数字地 (DGND) 在 LDO 输出电容处单点连接（星型接地），不要在多个地方桥接两地。

#### 2. 为什么 NRF24L01 而不是 ESP32 WiFi

- NRF24L01 + PA + LNA 模块功耗低（发射 ~12mA，接收 ~14mA），适合电池供电
- SPI 接口简单，不占用 UART
- 配合另一个 NRF24 做 USB dongle → 无线调试/遥控，这是面试加分项
- 如果确实需要 WiFi/BLE，可以在排针上预留 ESP32 模块的接口

#### 3. CAN 终端电阻

SN65HVD230 CAN 收发器需要在 CAN_H 和 CAN_L 之间加 120Ω 终端电阻。**只在总线两端加**，中间节点不加。

板子上留一个 120Ω 0805 电阻位置 + 跳线帽：
- 作为总线端点时：插上跳线帽，接入 120Ω
- 作为中间节点时：拔掉跳线帽

#### 4. 电机驱动的续流二极管

DRV8870 内置了 H 桥的 MOSFET 和体二极管，不需要外加续流二极管。但 **VM 引脚（电机电源）的大电容（100µF + 100nF）必须靠近芯片**，因为电机 PWM 切换时从电源抽取脉冲电流。

### Layout 分区策略

```
┌─────────────────────────────────────────────────────────┐
│  [电源区]              [MCU + 数字区]         [接插件区] │
│  USB Type-C             STM32F103            排针引出    │
│  AMS1117                CH340C                SWD 接口   │
│  LP2985 (模拟LDO)       去耦电容×4            编码器接口  │
│                                                     │
│  [电机区]              [无线区]              [CAN 区]  │
│  DRV8870               NRF24L01              SN65HVD230│
│  大电容 100µF           PCB 天线区域          CAN 端子  │
│  电源隔离                                        │
└─────────────────────────────────────────────────────────┘

关键隔离原则：
- 电机区 → MCU 数字区之间用地槽隔离（~2mm 开槽 + 单点桥接）
- NRF24L01 天线区域下方不铺铜（避免天线 detuning）
- CAN 走线做差分对（等长、等距），尽量远离 PWM 走线
```
