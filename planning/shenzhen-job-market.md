## 一、常见招聘要求汇总

### 1.1 嵌入式软件工程师（初级/应届 0-2年）

| 硬件基础 | 能看懂原理图，会使用示波器/逻辑分析仪/万用表 |

## 三、最热门技能与需求分析

### 3.1 RTOS 需求排行

| RTOS | 热度 | 应用领域 | 学习建议 |
|------|------|---------|---------|
| **FreeRTOS** | ★★★★★ | 全行业通用（IoT/消费电子/工业） | **必学**，开源免费，生态最大 |

## 五、面试常见题目与考点

### 5.1 C语言基础（必考，占面试30-40%）

1. **static/const/volatile/extern 关键字** - 含义、用法、区别
2. **指针与数组** - 指针运算、函数指针、指针数组 vs 数组指针
3. **结构体内存对齐** - `#pragma pack`、`__attribute__((packed))`
4. **位操作** - 置位、清零、取反、移位，寄存器操作实战
5. **内存管理** - 栈 vs 堆、malloc/free、内存泄漏检测
6. **中断函数** - ISR写法、volatile变量、中断嵌套
7. **联合体(Union)** - 大小端判断、寄存器位域访问
8. **宏定义 vs 内联函数** - 优缺点，`do-while(0)`技巧
9. **回调函数** - 函数指针实现、注册机制
10. **链表/队列/环形缓冲区** - 嵌入式常用数据结构

### 5.2 单片机与RTOS（占面试20-30%）

1. **STM32启动流程** - 从复位向量到main函数的全过程
2. **中断优先级/NVIC** - 抢占优先级、子优先级、中断嵌套
3. **时钟树配置** - HSI/HSE/PLL，系统时钟和外设时钟
4. **DMA使用** - 什么场景用，双缓冲/乒乓缓冲
5. **裸机 vs RTOS** - 何时该用RTOS，何时裸机足够
6. **任务调度** - 优先级反转、互斥锁/信号量/消息队列
7. **FreeRTOS任务状态** - Ready/Running/Blocked/Suspended
8. **Tickless低功耗** - 原理、实现
9. **任务栈大小估算** - 如何确定任务栈够用
10. **Bootloader设计** - 固件升级/OTA流程

### 5.3 硬件基础（占面试15-20%）

1. **上拉/下拉电阻** - 为什么需要，阻值选择
2. **GPIO模式** - 推挽/开漏，什么时候用哪个
3. **电平转换** - 3.3V/5V互连方案
4. **去耦电容** - 作用、布局原则
5. **电源拓扑** - LDO vs DC-DC，选型考量
6. **看门狗** - 独立看门狗 vs 窗口看门狗
7. **PCB Layout** - 高速信号布线规则（差分信号、阻抗匹配）

### 5.4 通信协议（占面试10-15%）

1. **I2C时序** - 起始/停止条件、应答、时钟拉伸
2. **SPI四种模式** - CPOL/CPHA区别
3. **UART波特率** - 误差容忍度、如何配置
4. **CAN总线** - 仲裁机制、错误帧、CAN FD vs CAN 2.0
5. **MQTT QoS** - QoS 0/1/2区别与使用场景

### 5.5 项目经验与系统设计（占面试10-20%）

1. **深挖简历项目** - 你在项目中具体做了什么，遇到什么问题如何解决
2. **系统设计题** - "设计一个智能门锁的嵌入式方案"（选MCU、传感器、通信、功耗估算）
3. **Debug场景** - "设备偶发死机，如何排查？"
4. **团队协作** - Git工作流、代码review经验

### 5.6 手写代码题（常见）

```
// 经典题1：反转字符串
void reverse_string(char *str);

// 经典题2：大小端判断

// 经典题3：设置寄存器某位
#define SET_BIT(reg, bit)    ((reg) |= (1 << (bit)))
#define CLEAR_BIT(reg, bit)  ((reg) &= ~(1 << (bit)))
#define TOGGLE_BIT(reg, bit) ((reg) ^= (1 << (bit)))
#define GET_BIT(reg, bit)    (((reg) >> (bit)) & 0x01)

// 经典题4：环形缓冲区实现
typedef struct {
    uint8_t *buffer;
    uint32_t head;
    uint32_t tail;
    uint32_t size;
} ring_buffer_t;

// 经典题5：软件定时器/延时
```

## 六、针对用户背景的学习路径建议

### 6.1 当前优势分析

| 现有经验 | 价值评估 | 如何利用 |
|---------|---------|---------|
| STM32经验 | ★★★★★ | 核心优势，继续深入FreeRTOS+STM32项目 |
| 追觅科技（扫地机器人） | ★★★★☆ | 行业经验宝贵，内部转岗或跳槽同类公司 |
| 测试工程师背景 | ★★★☆☆ | 可强调质量意识、Debug能力、系统性思维 |
| ROS2基础 | ★★★☆☆ | 机器人行业加分，结合嵌入式底层 |
| 逻辑分析仪/DP100 | ★★★★☆ | 硬件调试能力，嵌入式必备 |
| 各种传感器 | ★★★★☆ | ToF/摄像头驱动经验直接可迁移 |
| Raspberry Pi 5/Arduino | ★★☆☆☆ | 原型能力，但面试价值有限 |

### 6.2 学习路线（从现在到2026年4月，约10个月）

**Phase 1：夯实基础（1-2个月）**
- C语言深度复习：指针、内存、数据结构（推荐《C和指针》《C专家编程》）
- STM32外设全面梳理：定时器/PWM/ADC/DMA/中断优先级

## 七、关键建议与策略

### 7.1 求职策略

1. **内部转岗优先**：追觅科技内部转嵌入式是最稳妥路径，你的产品理解是新人没有的优势
2. **扫地机器人生态**：追觅 -> 科沃斯/石头/云鲸/小米生态，行业经验直接对口
3. **深圳机器人公司**：大疆/普渡/海柔/越疆/优必选，ROS2+STM32组合很吃香
4. **警惕培训骗局**：嵌入式培训是重灾区，不要花几万块报班，自学完全够

### 7.2 面试技巧

1. **测试经验包装**：不说"我是测试"，说"精通嵌入式系统验证、熟悉软硬件联调，有丰富的Debug经验"
2. **项目深挖**：准备2-3个深入的项目，能讲清楚架构设计、问题排查过程
3. **示弱策略**：坦诚说"我还在学习XX，但有扎实的YY基础，可以快速上手"

### 7.3 关键时间节点

- 2026年春节后（2-3月）：招聘旺季，大量岗位放出
- 2026年3-4月：你的目标入职窗口
- 提前1-2个月开始投递和面试
