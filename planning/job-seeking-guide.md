## 三、你的差异化武器：408 基础

大多数嵌入式候选人来自电子/自动化背景，很少系统学过计算机核心课程。你备考过 408（数据结构/组成原理/操作系统/计算机网络），这在面试中是**隐藏的降维打击**。

### 面试中自然带出 408

关键不是你"学过 408"，而是用 408 的理论解释嵌入式问题——这让面试官觉得你不是"调库工程师"，而是"理解系统的人"。

**场景 1：被问环形缓冲区**
> "这和循环队列的判空判满条件一样—— `(rear+1) % SIZE == front`。我写 UART DMA 双缓冲时直接用了这个思想。"

**场景 2：被问 FreeRTOS 调度原理**
> "FreeRTOS 的抢占式调度就是操作系统的优先级调度——每个优先级一个就绪链表，用位图做 O(1) 查找最高优先级任务。这和传统 OS 的多级反馈队列是同一个思想，只是 FreeRTOS 更轻量。"

**场景 3：被问为什么用双向链表管理任务**
> "任务频繁在就绪/阻塞/挂起状态间切换，双向链表插入删除是 O(1)，不需要连续内存。如果换成数组，每次任务状态变更都要 O(n) 搬移，MCU 内存也撑不住。"

### 408 面试速查表

| 408 概念 | 嵌入式对应 | 面试怎么用 |
|----------|----------|-----------|
| 循环队列 | UART 环形缓冲区 | 手写环形缓冲区时直接搬公式 |
| 双向链表 | FreeRTOS 就绪任务链表 | 解释内核为什么选链表不选数组 |
| 大小端 + 内存对齐 | STM32 寄存器访问 | C 语言题秒答 |
| 进程调度 | FreeRTOS 任务调度 | 从 OS 理论角度讲抢占式调度 |
| PV 操作 / 信号量 | FreeRTOS 互斥量/信号量 | 理解 IPC 选择的理论依据 |
| 死锁条件 | 优先级反转问题 | 解释为什么互斥量需要优先级继承 |
| 中断响应 | ARM NVIC 中断处理 | 能画出中断响应流程 |
| 网络分层 | CAN/MQTT 协议栈 | 用网络思维理解嵌入式通信 |

## 四、面试话术：TSE→嵌入式 最关键的 5 个回答

### 3.1 "你为什么从测试转嵌入式？"

❌ 不要说："测试没前途" / "我想学技术"

✅ 这样说：

> "我在追觅做 TSE，日常需要分析和复现嵌入式固件的 bug。为了能自己定位问题根因而不是只提单等开发改，我开始自学 STM32——从 GPIO、I2C 驱动开始，慢慢理解了固件是怎么控制传感器和电机的。这个过程让我发现自己对底层更感兴趣。我系统学过 408 计算机基础，学嵌入式的时候发现很多概念是通的——比如 FreeRTOS 调度就是 OS 进程调度的微缩版，学起来比纯硬件背景的人快。"

### 3.2 "你没有嵌入式工作经验，怎么胜任？"

❌ 不要说："我可以学"

✅ 这样说：

> "我确实没有在公司做过嵌入式岗位，但我在业余时间从寄存器级写了 STM32 的 GPIO、I2C/OLED 驱动，不是调 HAL 库。同时我系统学过 408——数据结构、操作系统、组成原理、网络——这些让我能比别人更快理解 FreeRTOS 的调度机制、DMA 的总线原理、环形缓冲区的数据结构选型。再加上我在追觅一年积累的对嵌入式 bug 模式的理解，我知道量产代码容易在哪些地方出问题。缺的是商业项目经验，但学习成本比纯新人低。"

### 3.3 "你最大的优势是什么？"

✅ 这样说：

> "我的优势是结合 408 计算机基础去理解嵌入式。比如学 FreeRTOS 任务调度的时候，我直接对应到 OS 的优先级调度模型；学 DMA 的时候对应到总线和中断控制器的概念。这比单纯背寄存器配置要深入。加上我在追觅做 TSE，每天都在接触嵌入式固件的问题复现——中断没清标志位导致死机、环形缓冲区溢出覆盖内存、I2C 时序不对传感器读不到数——这些 bug 模式我都见过，以后自己写代码的时候会有意识地避免。"

### 3.4 "你期望薪资多少？"

✅ 底线策略：

> "我目前月薪 13.5K + 年终约 3 个月。考虑到我有嵌入式基础（STM32 寄存器编程 + 408 理论）和扫地机器人行业经验，期望 16-20K。我也理解转岗有个过渡期，可以根据岗位匹配度谈。"

### 3.5 "你做过最难的嵌入式项目是什么？"

✅ 诚实回答（两个版本，选一个）：

**版本 A：如果你线激光 3D 扫描确实是自己做的**

> "我用 650nm 线激光 + USB 摄像头做了一个 3D 扫描项目。从相机标定、激光线提取、到 Open3D 点云重建的完整流程。最大的挑战是激光线提取——灰度重心法在强光下误差大，后来用 Steger 算法做亚像素提取，精度从 ±0.5mm 提到 ±0.05mm。"

**版本 B：如果你线激光项目也没深入做**

> "我目前在用 STM32 做一个学习项目——从寄存器级裸机编程开始，GPIO → I2C OLED 驱动 → 现在在做定时器 PWM 和 PID 速度控制。虽然项目规模不大，但每个模块我都是先看芯片手册、用逻辑分析仪验证时序、再写代码。比如 I2C 驱动——我能画出来从写数据寄存器到 SCL 引脚出时钟的完整时序。408 里学过总线协议，实际用逻辑分析仪抓到波形的时候，理论和实践就对上了。"

## 六、练手面试的黄金法则

### 6.1 练手阶段的目标

- 🎯 目标 1：习惯嵌入式面试的节奏和题型

### 6.2 练手面试后必做

每次面试后 30 分钟内记录：
1. 被问了哪些题？（特别是答不上来的）
2. 我的"转岗故事"对方反应如何？（需要调整吗）
3. 对方公司用的什么技术栈？（市场信号！）

### 6.3 练手→目标的过渡信号

当你连续 3 场面试能做到：
- 技术问题 80% 流畅回答
- 转岗故事讲完对方点头/追问（不是质疑）
- 项目介绍时对方能追问细节（说明感兴趣）

→ 可以切到第二阶段目标公司了。

> "感谢认可。我现在追觅年包大概 21 万左右。考虑到我有扫地机器人行业经验、自学的嵌入式技术栈可以直接上手项目，期望 20K 以上。如果 base 能到 XX，我可以一周内确认。"

---

## 八、关键提醒

1. **不要裸辞。** 在职骑驴找马，拿到书面 offer 再提离职。
2. **追觅内部先探路。** 找嵌入式部门的同事聊聊，问问有没有内部转岗机会。这是最稳的路径。
3. **不要被"3-5 年经验"拦住。** 很多 JD 写 3-5 年但实际面试发现你项目经验扎实也会要。第一阶段练手时大胆投。
4. **GitHub 就是你的作品集。** 保持 commit 活跃，README 写清楚，面试时直接给链接。
5. **面试是概率游戏。** 面 10 家拿 2 个 offer 是正常水平，不要因为挂了几家就自我怀疑。
