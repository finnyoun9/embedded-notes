> 目标：2026 年 9-10 月从追觅 TSE 跳槽深圳嵌入式工程师

> 编写日期：2026-06-14

---

## 一、求职策略总纲

### 1.1 三段式投递法

不要海投。按三个阶段递进：

```
第一阶段：练手（10-15 家）  → 积累面试经验，不期待 offer
第二阶段：目标（10 家）      → 真正想去的公司，充分准备后投
第三阶段：冲刺（5 家）       → 最想去的，用前两阶段经验拿下
```

### 1.2 时间线

```
6月         7月           8月           9月            10月
├─ 简历打磨  ├─ 第一阶段投递 ├─ 第二阶段投递 ├─ 第三阶段冲刺 ├─ 入职
├─ 八股文    ├─ 练手面试    ├─ 目标面试     ├─ offer 谈判
├─ 项目收尾  ├─ 简历迭代    ├─ 项目补强     └─ 决策
```

## 三、面试话术：TSE→嵌入式 最关键的 5 个回答

### 3.1 "你为什么从测试转嵌入式？"

❌ 不要说："测试没前途" / "我想学技术"

✅ 这样说：

> "我在追觅做 TSE 的过程中，为了分析扫地机器人的传感器数据和电机控制逻辑，自学了 STM32 裸机编程和 I2C/SPI 驱动开发。做了几个项目后发现嵌入式才是我真正想做的方向——我享受从寄存器级理解硬件、用 C 语言控制外设的过程。我在 TSE 岗位积累的对扫地机器人整机系统的理解、跨部门协调能力、和 Debug 方法论，反而是纯嵌入式新人没有的优势。"

### 3.2 "你没有嵌入式工作经验，怎么胜任？"

❌ 不要说："我可以学"

✅ 这样说：

> "我确实没有在公司做过嵌入式岗位，但我在业余时间用 STM32F103 完成了 4 个裸机驱动项目和 1 个 FreeRTOS 多任务项目——不是跑 demo，是从寄存器配置、时钟树、链接脚本全部自己写。同时我做了完整的 BLDC 电机六步换向控制 + 速度 PID，用逻辑分析仪验证了 PWM 波形。这些项目的代码都在我的 GitHub 上，和 1 年经验的嵌入式工程师比，我缺的不是技术能力，是商业项目的流程经验，但这个我可以快速补上。"

### 3.3 "你最大的优势是什么？"

✅ 这样说：

> "我的优势是软硬结合的视角。在追觅做 TSE，我每天都在分析和复现嵌入式软件的 bug——我知道什么样的代码会在产线上出问题、什么样的设计会导致客诉。这种'从产品反推技术'的思维，是纯做嵌入式开发的人很难获得的。再加上我自学的 STM32/FreeRTOS/ROS2 技术栈，我现在能独立完成从 MCU 驱动到上位机联调的全链路。"

### 3.4 "你期望薪资多少？"

✅ 底线策略：

> "我目前月薪 13.5K + 年终约 3 个月。考虑到我从 TSE 转嵌入式的技能积累（STM32/FreeRTOS/BLDC/ROS2），以及我对扫地机器人行业的深度理解，期望 18-22K。当然我也理解转岗的过渡期，可以根据岗位匹配度调整。"

### 3.5 "你做过最难的嵌入式项目是什么？"

✅ 必讲项目（选 1 个深入讲）：

**首选：BLDC 电机六步换向 + PID 速度控制**

> "我用 STM32F103 的 TIM1 高级定时器产生互补 PWM 驱动无刷电机，通过 Hall 传感器读取转子位置实现六步换向。遇到的问题：高速时换相噪声大，后来发现是 ADC 采样时机不对——换相瞬间电流尖峰会干扰采样。我用逻辑分析仪抓到换相噪声窗口，在 PWM OFF 期间采样，噪声从 ±30% 降到 ±3%。最后实现了速度 PID 闭环控制，目标 3000 RPM 稳态误差 < 2%。"

**备选：线激光 3D 扫描系统**

> "我用 650nm 线激光 + USB 摄像头实现了激光三角测量，从标定、图像处理、到 Open3D 点云重建的全 pipeline。最大挑战是线激光提取——直接用灰度重心法误差大，后来换成 Steger 算法做亚像素提取，精度从 ±0.5mm 提到 ±0.05mm。"

## 五、面试准备清单

### 5.1 第一轮面试前必须完成

- [ ] C 语言 20 道高频题能对答（static/volatile/const/指针/位操作/内存对齐）
- [ ] 能画出 STM32 启动流程（向量表→SystemInit→main）
- [ ] 能画出 FreeRTOS 任务状态转换图
- [ ] 能讲清楚 I2C/SPI/UART 时序差异和适用场景
- [ ] 准备 2 个可以深入讲的 STAR 项目故事
- [ ] 简历上每个字都能被追问（别写"了解"但一问三不知）

### 5.2 面试中必问的 5 类问题

详细内容见 `planning/interview-prep-八股文.md`，这里列最高频的：

| 类别 | 最高频 | 准备优先级 |
|------|--------|----------|
| C 语言 | volatile 作用、static 三种用法、指针数组 vs 数组指针 | 🔴 P0 |
| STM32 | 启动流程、中断优先级、时钟树、GPIO 模式 | 🔴 P0 |
| FreeRTOS | 任务调度、队列 vs 信号量、优先级反转 | 🔴 P0 |
| 通信协议 | I2C 时序、SPI 四种模式、CAN 仲裁 | 🟡 P1 |
| 项目 | "你最难的问题是什么？怎么解决的？" | 🔴 P0 |

### 5.3 反问环节（必须问！显示你有思考）

✅ 好问题：
- "团队目前用的是什么 MCU 平台和 RTOS？"
- "这个岗位主要负责哪个模块？驱动层还是应用层？"
- "团队的技术栈是裸机还是 RTOS？有计划上 Linux 吗？"
- "新人入职的 onboarding 流程是怎样的？"

❌ 不问：
- "加班多吗？"（可以侧面问"项目节奏怎么样"）
- "薪资还能谈吗？"（等 offer 阶段再谈）

## 六、练手面试的黄金法则

### 6.1 练手阶段的目标

- 🎯 目标 1：习惯嵌入式面试的节奏和题型
- 🎯 目标 2：打磨"TSE→嵌入式"这个转岗故事的讲法
- 🎯 目标 3：发现自己的技术盲区，回来补
- 🎯 目标 4：拿到 1-2 个 offer（即使不去，有 offer 谈薪有底气）

### 6.2 练手面试后必做

每次面试后 30 分钟内记录：
1. 被问了哪些题？（特别是答不上来的）
2. 我的"转岗故事"对方反应如何？（需要调整吗）
3. 对方公司用的什么技术栈？（市场信号！）

### 6.3 练手→目标的过渡信号

当你连续 3 场面试能做到：
- 技术问题 80% 流畅回答
- 转岗故事讲完对方点头/追问（不是质疑）
- 项目介绍时对方能追问细节（说明感兴趣）

→ 可以切到第二阶段目标公司了。

## 八、关键提醒

1. **不要裸辞。** 在职骑驴找马，拿到书面 offer 再提离职。
2. **追觅内部先探路。** 找嵌入式部门的同事聊聊，问问有没有内部转岗机会。这是最稳的路径。
3. **不要被"3-5 年经验"拦住。** 很多 JD 写 3-5 年但实际面试发现你项目经验扎实也会要。第一阶段练手时大胆投。
4. **GitHub 就是你的作品集。** 保持 commit 活跃，README 写清楚，面试时直接给链接。
5. **面试是概率游戏。** 面 10 家拿 2 个 offer 是正常水平，不要因为挂了几家就自我怀疑。
