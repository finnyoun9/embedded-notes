# 嵌入式面试冲刺学习计划 — 2026.8-10 投递窗口

> 基于 shenzhen-job-market.md 调研报告，按面试出题频率和简历说服力排优先级。
> 原则：**只练面试会考的，只做简历能讲的。** 不搞研究型学习。

---

## 优先级法则

```
所有技能分三级：

🔴 P0 — 面试必考，不会直接挂 → 6-7月死磕
🟡 P1 — 简历亮点，拉开差距   → 7-8月集中出成果
🟢 P2 — 锦上添花，有时间就做  → 9月碎片时间
❌ 砍掉 — 面试不考、简历上写了还减分 → 不做
```

---

## 🔴 P0：面试必考（6 月-7 月中旬，每天 1-2h）

**目标**：任何一场面试，技术面能撑过前 15 分钟。

### C 语言（占面试 40%，最高频）

| 主题 | 具体内容 | 优先级 | 时间 |
|------|---------|--------|------|
| 关键字 | static/const/volatile/extern — 每种至少 3 个使用场景 | 🔴🔴🔴 | 6月下旬 |
| 指针 | 指针数组 vs 数组指针、函数指针、回调函数 | 🔴🔴🔴 | 6月下旬 |
| 位操作 | 置位/清零/取反/移位宏、寄存器操作实战 | 🔴🔴🔴 | 6月下旬 |
| 内存 | 栈 vs 堆、结构体内存对齐、大小端判断 | 🔴🔴 | 7月上旬 |
| 数据结构 | 环形缓冲区（手写）、链表反转 | 🔴🔴 | 7月上旬 |
| 手写代码 | 反转字符串、大小端判断、环形缓冲区 | 🔴🔴 | 7月上旬 |

**验证标准**：牛客网嵌入式 C 语言专项刷完 50 题，正确率 > 85%。

### STM32 + ARM（占面试 25%）

| 主题 | 具体内容 | 时间 |
|------|---------|------|
| 启动流程 | 向量表 → SystemInit → __main → main，能画出来 | 7月上旬 |
| 中断/NVIC | 抢占优先级 vs 子优先级，中断嵌套规则 | 7月上旬 |
| 时钟树 | HSI/HSE/PLL 配置路径，系统时钟和外设时钟关系 | 7月上旬 |
| GPIO | 推挽/开漏/上下拉，什么时候用哪个 | 7月上旬 |
| DMA | 什么场景用，双缓冲原理 | 7月中旬 |
| 看门狗 | 独立看门狗 vs 窗口看门狗 | 7月中旬 |

**验证标准**：能对着白板画出 STM32 启动流程和时钟树。

### FreeRTOS（占面试 20%）

| 主题 | 具体内容 | 时间 |
|------|---------|------|
| 任务调度 | 任务状态转换图（Ready/Running/Blocked/Suspended） | 7月中旬 |
| IPC | 队列 vs 信号量 vs 互斥量 vs 事件组，各自使用场景 | 7月中旬 |
| 优先级反转 | 什么情况发生，怎么解决（互斥量优先级继承） | 7月中旬 |
| 中断管理 | ISR 中能用哪些 API（FromISR 版本） | 7月中旬 |
| 任务栈 | 如何估算任务栈大小 | 7月中旬 |

**验证标准**：能讲清楚"5 个任务并行采集传感器"这个项目的架构和 IPC 选择。

### 通信协议（占面试 15%）

| 协议 | 具体内容 | 时间 |
|------|---------|------|
| I2C | 时序图（起始/停止/应答/时钟拉伸），7 位地址 | 7月中旬 |
| SPI | 4 种模式（CPOL/CPHA），全双工，无流控 | 7月中旬 |
| UART | 波特率误差容忍，TTL vs RS232 电平 | 7月中旬 |
| CAN | 仲裁机制、数据帧格式、CAN 2.0 vs CAN FD | 7月中旬 |

**验证标准**：能对比 I2C/SPI/UART 的优劣和适用场景，能画 I2C 时序图。

## 🟡 P1：简历亮点（7 月中旬-8 月底，周末集中）

**目标**：简历上有 2-3 个面试官会追问的项目。

### ⭐ 最高优先级：BLDC 电机控制实物

为什么这是 P1 中的 P0：
- 面试能讲"我用 STM32 驱动了无刷电机" — 秒杀 90% 竞争者
- 能用逻辑分析仪抓 PWM 波形展示 — 实物证据
- 几乎所有机器人工厂都看重电机控制经验

| 里程碑 | 内容 | 证明方式 | 时间 |
|--------|------|---------|------|
| BLDC-1 | 六步换向让电机转起来 | 实物视频 + 逻辑分析仪 6 路 PWM 波形 | 8月上旬 |
| BLDC-2 | Hall 传感器读取 + 速度计算 | 串口打印实时转速 | 8月中旬 |
| BLDC-3 | 速度 PID 闭环 | 目标 3000RPM ±2%，录视频 | 8月下旬 |

### ⭐ 简历项目打磨（每个项目准备 3 分钟版本和 10 分钟版本）

| 项目 | 讲什么 | 面试必问 |
|------|--------|---------|
| BLDC 电机控制 | TIM1 互补 PWM + Hall 换向 + PID | "换相噪声怎么解决的？" |
| FreeRTOS 多任务采集 | 5 任务/队列/信号量架构 | "为什么用队列不用信号量？" |
| 线激光 3D 扫描 | 标定→Steger 提取→点云 | "精度怎么验证的？" |

### 补充技能

| 技能 | 学到什么程度 | 时间 |
|------|-------------|------|
| MQTT | 能在 ESP32 上跑通 MQTT 发布订阅 | 8月碎片 |
| Python 上位机 | pyserial + matplotlib 实时画传感器数据 | 8月碎片 |
| Git | 会 branch/rebase/PR 流程 | 日常 |

这些东西面试偶尔问到，但性价比低——不学也不会挂。

| 内容 | 投入 | 产出 | 建议 |

## 6 月→9 月 周计划

```
【6月下旬 — P0 死磕期】
Week 1-2: C 语言 20 道高频题 + 手写代码题（环形缓冲区、字符串反转、大小端）
Week 3:   STM32 启动流程 + 中断/NVIC + 时钟树（画图）
Week 4:   FreeRTOS 任务/IPC 理论 + 通信协议对比表

验证：牛客网 C 语言 50 题刷完

【7月 — P0 收尾 + P1 启动】
Week 1:   通信协议（I2C/SPI/UART/CAN 对比 + 时序图）
Week 2:   BLDC-1：六步换向让电机转起来（最重要！）
Week 3:   BLDC-2：Hall 传感器 + 速度计算 + 逻辑分析仪验证
Week 4:   简历初稿完成 + BOSS直聘挂出（练手阶段）

验证：简历写完，能对着镜子讲完 3 分钟自我介绍 + 1 个项目

【8月 — P1 冲刺 + 练手面试】
Week 1:   BLDC-3：速度 PID 闭环调通
Week 2:   简历迭代 v2 + 第一阶段投递开始（普渡/海柔/拓斯达）
Week 3:   练手面试 + 每次面试后记录盲区 + 补漏
Week 4:   练手面试 + MQTT/Python 碎片补

验证：至少面完 3 家练手，每次能完整讲转岗故事

【9月 — 目标公司冲刺】
Week 1:   复盘练手面试，针对盲区补强
Week 2:   第二阶段投递（科沃斯/石头/云鲸/优必选/越疆）
Week 3:   目标面试 + BLDC 项目视频完善
Week 4:   目标面试 + 谈 offer

验证：拿到至少 1 个目标公司 offer

【10月 — 收尾】
Week 1:   第三阶段冲刺（大疆/乐鑫，如果有信心）
Week 2:   offer 决策 + 提离职
```

## ❌ 明确砍掉的内容

以下东西在 8-10 月面试窗口期内 **不值得花时间**：

| 内容 | 砍掉原因 |
|------|---------|
| 嵌入式 Linux 驱动/设备树 | 2 个月学不到面试能讲的程度 |
| ROS2 深入（SLAM/Nav2） | 只有扫地机器人公司问，概率 < 10% |
| RISC-V | 面试几乎不考 |
| TinyML / 边缘 AI | 极少岗位要求，性价比为 0 |
| Rust 嵌入式 | 国内招聘市场几乎不提 |
| KiCad 画多层板 | 你不是面硬件工程师 |
| YOLO / OpenCV 深入 | 除非面感知岗，否则不考 |

## 每日时间分配建议

```
工作日（每天 2h）：
  1h  — P0 理论（C 语言/STM32/FreeRTOS/协议）
  0.5h — 刷面试题（牛客网/手写代码）
  0.5h — 项目推进（BLDC）

周末（每天 4h）：
  2h  — BLDC 项目实物（最重要！）
  1h  — 项目文档 + GitHub 更新
  1h  — 简历迭代 / 复盘 / 查漏补缺
```

**核心原则**：BLDC 项目是第一优先级。理论刷再多，不如一个能展示的视频有说服力。
