## 🎓 你的隐藏王牌：408 基础

大多数嵌入式候选人的背景是电子/自动化，**系统学过计算机 408 四门的人很少**。你的 408 基础不是"过去时"，而是面试中的差异化武器——嵌入式本质上是"资源受限的计算机系统"，408 里几乎每个核心概念都能在嵌入式里找到直接映射。

> 详细映射见 [cs408-to-embedded-mapping.md](cs408-to-embedded-mapping.md)，以下聚焦面试怎么用。

### 408 → 嵌入式面试 四大映射线

| 408 科目 | 嵌入式面试映射 | 面试场景 |
|----------|---------------|---------|
| **数据结构** | 环形缓冲区 = 循环队列（判空/判满条件一模一样）<br>FreeRTOS 任务调度 = 双向链表 + 优先级数组<br>中断向量表 = 哈希表（O(1) 查找） | "手写一个环形缓冲区" ← 这就是 408 循环队列题 |
| **计算机组成** | ARM Cortex-M3 = 408 学过的 CPU 核心概念<br>（流水线、哈佛架构、中断响应、DMA 总线）<br>大小端判断、内存对齐 = 408 原题 | "从 Reset_Handler 到 main 发生了什么？" ← 组成原理 + 启动代码 |
| **操作系统** | FreeRTOS = 408 操作系统的微缩版<br>任务调度 = 进程调度（优先级抢占）<br>信号量/互斥量 = PV 操作原题<br>优先级反转 = 408 的经典考点 | "FreeRTOS 任务调度的原理是什么？" ← 你可以从 OS 调度算法讲起 |
| **计算机网络** | CAN 总线仲裁 = CSMA/CD 的变体<br>MQTT QoS = TCP 可靠传输的简化版<br>UART/SPI/I2C = 物理层 + 链路层 | "CAN 总线为什么不需要主机？" ← 网络拓扑思维 |

### 面试中怎么自然带出 408

❌ 不要生硬地说"我学过 408"
✅ 在回答问题时自然融入：

> **被问环形缓冲区**："这和循环队列的判空判满条件一样—— `(rear+1) % SIZE == front`。我在 408 数据结构里系统学过队列，所以在写 UART DMA 双缓冲的时候，直接用了循环队列的思想，省了 200 字节内存。"

> **被问 FreeRTOS 调度**："FreeRTOS 的抢占式调度本质上就是操作系统的优先级调度——高优先级任务就绪时立即抢占低优先级任务，和 Linux CFS 的 vruntime 调度、或者传统 OS 的多级反馈队列是一个思路，只是 FreeRTOS 用的是每个优先级一个就绪链表，查找是 O(1) 的位图算法。"

> **被问为什么选双向链表管理任务**："因为任务会频繁在不同状态间切换——从就绪链表摘下来、插入阻塞链表。双向链表插入删除是 O(1)，而且不需要连续内存。如果用的是数组，每次任务状态切换都要整体搬移，在 MCU 有限内存下不可接受。"

### 408 复习优先级（对着面试看）

不需要重新复习全部 408——只需要激活和面试直接相关的那部分：

| 408 知识点 | 为什么复习 | 用时 |
|-----------|----------|------|
| 循环队列判空/判满 | 环形缓冲区手写题，最高频 | 15 分钟 |
| 双向链表插入/删除 | 理解 FreeRTOS 任务调度内核 | 15 分钟 |
| 大小端 + 内存对齐 | C 语言面试必考题 | 10 分钟 |
| 进程调度（抢占/非抢占） | FreeRTOS 调度的理论根基 | 20 分钟 |
| PV 操作 / 信号量 | FreeRTOS IPC 的理论根基 | 20 分钟 |
| 死锁四个条件 | 优先级反转问题的延伸讨论 | 10 分钟 |
| 中断响应过程（保存现场） | STM32 中断处理流程 | 15 分钟 |
| TCP 可靠传输 / 滑动窗口 | MQTT QoS 机制的理论背景 | 10 分钟 |

**总计：不到 2 小时**就能把 408 中和嵌入式面试最相关的部分激活。

## 🔴 P0：面试必考（6 月-7 月中旬，每天 1-2h）

**目标**：任何一场面试，技术面能撑过前 15 分钟。

### C 语言（占面试 40%，最高频）

| 主题 | 具体内容 | 优先级 | 时间 |
|------|---------|--------|------|
| 关键字 | static/const/volatile/extern — 每种至少 3 个使用场景 | 🔴🔴🔴 | 6月下旬 |
| 指针 | 指针数组 vs 数组指针、函数指针、回调函数 | 🔴🔴🔴 | 6月下旬 |
| 位操作 | 置位/清零/取反/移位宏、寄存器操作实战 | 🔴🔴🔴 | 6月下旬 |
| 内存 | 栈 vs 堆、结构体内存对齐、大小端判断 | 🔴🔴 | 7月上旬 |
| 数据结构 | 环形缓冲区（手写）、链表反转 | 🔴🔴 | 7月上旬 |
| 手写代码 | 反转字符串、大小端判断、环形缓冲区 | 🔴🔴 | 7月上旬 |

**验证标准**：牛客网嵌入式 C 语言专项刷完 50 题，正确率 > 85%。

### STM32 + ARM（占面试 25%）

| 主题 | 具体内容 | 时间 |
|------|---------|------|
| 启动流程 | 向量表 → SystemInit → __main → main，能画出来 | 7月上旬 |
| 中断/NVIC | 抢占优先级 vs 子优先级，中断嵌套规则 | 7月上旬 |
| 时钟树 | HSI/HSE/PLL 配置路径，系统时钟和外设时钟关系 | 7月上旬 |
| GPIO | 推挽/开漏/上下拉，什么时候用哪个 | 7月上旬 |
| DMA | 什么场景用，双缓冲原理 | 7月中旬 |
| 看门狗 | 独立看门狗 vs 窗口看门狗 | 7月中旬 |

**验证标准**：能对着白板画出 STM32 启动流程和时钟树。

### FreeRTOS（占面试 20%）

| 主题 | 具体内容 | 时间 |
|------|---------|------|
| 任务调度 | 任务状态转换图（Ready/Running/Blocked/Suspended） | 7月中旬 |
| IPC | 队列 vs 信号量 vs 互斥量 vs 事件组，各自使用场景 | 7月中旬 |
| 优先级反转 | 什么情况发生，怎么解决（互斥量优先级继承） | 7月中旬 |
| 中断管理 | ISR 中能用哪些 API（FromISR 版本） | 7月中旬 |
| 任务栈 | 如何估算任务栈大小 | 7月中旬 |

**验证标准**：能讲清楚"5 个任务并行采集传感器"这个项目的架构和 IPC 选择。

### 通信协议（占面试 15%）

| 协议 | 具体内容 | 时间 |
|------|---------|------|
| I2C | 时序图（起始/停止/应答/时钟拉伸），7 位地址 | 7月中旬 |
| SPI | 4 种模式（CPOL/CPHA），全双工，无流控 | 7月中旬 |
| UART | 波特率误差容忍，TTL vs RS232 电平 | 7月中旬 |
| CAN | 仲裁机制、数据帧格式、CAN 2.0 vs CAN FD | 7月中旬 |

**验证标准**：能对比 I2C/SPI/UART 的优劣和适用场景，能画 I2C 时序图。

## 🟡 P1：简历亮点（7 月中旬-8 月底，周末集中）

**目标**：简历上有 2-3 个面试官会追问的项目。

### ⭐ 最高优先级：BLDC 电机控制实物

为什么这是 P1 中的 P0：
- 面试能讲"我用 STM32 驱动了无刷电机" — 秒杀 90% 竞争者
- 能用逻辑分析仪抓 PWM 波形展示 — 实物证据
- 几乎所有机器人工厂都看重电机控制经验

| 里程碑 | 内容 | 证明方式 | 时间 |
|--------|------|---------|------|
| BLDC-1 | 六步换向让电机转起来 | 实物视频 + 逻辑分析仪 6 路 PWM 波形 | 8月上旬 |
| BLDC-2 | Hall 传感器读取 + 速度计算 | 串口打印实时转速 | 8月中旬 |
| BLDC-3 | 速度 PID 闭环 | 目标 3000RPM ±2%，录视频 | 8月下旬 |

### ⭐ 简历项目打磨（每个项目准备 3 分钟版本和 10 分钟版本）

| 项目 | 讲什么 | 面试必问 |
|------|--------|---------|
| BLDC 电机控制 | TIM1 互补 PWM + Hall 换向 + PID | "换相噪声怎么解决的？" |
| FreeRTOS 多任务采集 | 5 任务/队列/信号量架构 | "为什么用队列不用信号量？" |
| 线激光 3D 扫描 | 标定→Steger 提取→点云 | "精度怎么验证的？" |

### 补充技能

| 技能 | 学到什么程度 | 时间 |
|------|-------------|------|
| MQTT | 能在 ESP32 上跑通 MQTT 发布订阅 | 8月碎片 |

## ❌ 明确砍掉的内容

以下东西在 8-10 月面试窗口期内 **不值得花时间**：

| 内容 | 砍掉原因 |
|------|---------|
| 嵌入式 Linux 驱动/设备树 | 2 个月学不到面试能讲的程度 |
| ROS2 深入（SLAM/Nav2） | 只有扫地机器人公司问，概率 < 10% |
| RISC-V | 面试几乎不考 |
| TinyML / 边缘 AI | 极少岗位要求，性价比为 0 |
| Rust 嵌入式 | 国内招聘市场几乎不提 |
| KiCad 画多层板 | 你不是面硬件工程师 |
| YOLO / OpenCV 深入 | 除非面感知岗，否则不考 |
