# STM32 扫地机器人学习指南

> 最后更新：2026-05-26

---

## 学习入口（从这里开始）

这份指南把 STM32 学习分成 **"基础 → 小车 → 扫地"** 三个阶段。你现在跟着**江科大教程 + 面包板**在补基础，后面再切到**好家伙VCC**做小车底盘，最后加上清扫逻辑变成扫地机器人。

**核心思路是按需学习**：不用把教程从头刷完，碰到什么需求就去对应章节找答案。下面这张表是你整个学习过程的总索引。

### 学习路径总览

```
阶段零：STM32 基础（江科大教程，面包板实验）
  → GPIO / 中断 / 定时器 / PWM / ADC / 通信协议
  → 约 50 集，但不需要全看——按需查下面的索引即可
  ↓
阶段一：STM32 智能小车基础（好家伙VCC）
  → 电机驱动、编码器、PID、MPU6050、避障、蓝牙
  ↓
阶段二：扫地机器人专项
  → 清扫机构、弓字形路径、状态机整合
  ↓
阶段三：进阶功能（选做）
  → WiFi/App、自动回充、栅格地图
```

### 江科大教程 vs 好家伙VCC 的关系

| 对比维度 | 江科大（阶段零） | 好家伙VCC（阶段一） |
|-|-|-|
| 定位 | 芯片外设基础，一个一个模块讲 | 项目驱动，围绕"做一台小车"展开 |
| 硬件 | 面包板 + 独立模块 | 小车底盘 + 电机 + 传感器集成 |
| 深度 | 每个外设讲透（寄存器、时序、多种用法） | 用到什么讲什么，重在整合 |
| 什么时候看 | **现在**，把基础模块跑通 | 基础模块熟悉后，开始做小车时看 |
| 重叠内容 | GPIO、中断、定时器、PWM、ADC、I2C、SPI | 电机驱动(PWM)、编码器(定时器)、MPU6050(I2C) |

> **建议节奏**：江科大先把 GPIO（P5-P8）、定时器/PWM（P13-P16）、编码器（P19-P20）看了，就可以切到好家伙VCC开始做小车。遇到通信协议的问题（I2C读MPU6050、串口接蓝牙），再回来补江科大的对应章节。

## 功能实现——按难度递进

### 第一级：底盘运动（最基础）

**目标**：走直线、转固定角度

```c
// PID 速度闭环 —— 扫地机器人能走直线的关键
void Motor_PID_Update(void) {
    int16_t speed_L = Encoder_Read(LEFT);
    int16_t speed_R = Encoder_Read(RIGHT);
    int16_t err = speed_L - speed_R;        // 差速 = 偏移量

    // 增量式 PI，10ms 调用一次
    int16_t adjust = Kp * err + Ki * err_integral;
    err_integral += err;
    if (err_integral > 500)  err_integral = 500;   // 积分限幅
    if (err_integral < -500) err_integral = -500;

    PWM_Set(LEFT,  target + adjust);
    PWM_Set(RIGHT, target - adjust);
}
```

> **知识准备**：江科大 P15-P16（PWM控制电机）→ P19-P20（编码器测速）→ 好家伙VCC 第7-9章（编码器+PID整定）

### 第二级：避障

**目标**：不撞墙、不摔下楼梯

```c
// 三级响应策略
if (ultra_center < 15.0f) {
    Robot_Stop();
    Robot_Backward(300ms);
    // 随机转向，防止卡死在角落
    (rand() & 1) ? Robot_TurnLeft(90) : Robot_TurnRight(90);
} else if (ultra_center < 30.0f) {
    // 减速并向更空旷的方向微调
    speed = SLOW_SPEED;
    if (ultra_left > ultra_right) yaw += 15;
    else                            yaw -= 15;
} else {
    speed = NORMAL_SPEED;   // 正常直行
}
```

> **知识准备**：江科大 P18（输入捕获测超声波脉宽）→ 好家伙VCC 第15章（超声波避障策略）

### 第三级：状态机

**目标**：让机器人"知道自己在干什么"

```c
typedef enum {
    STATE_IDLE,           // 待机
    STATE_CLEANING,       // 清扫直行
    STATE_AVOIDING,       // 避障后退
    STATE_TURNING,        // 转向中
    STATE_EDGE_FOLLOW,    // 沿墙清扫
    STATE_RETURNING,      // 回充
} State;

void FSM_Run(void) {
    State next = current;
    sensors = Read_All_Sensors();

    switch (current) {
    case STATE_CLEANING:
        if (sensors.cliff)      next = STATE_AVOIDING;
        if (sensors.obstacle)   next = STATE_AVOIDING;
        if (sensors.wall_left)  next = STATE_EDGE_FOLLOW;
        break;
    case STATE_AVOIDING:
        if (back_timeout())     next = STATE_TURNING;
        break;
    case STATE_TURNING:
        if (turn_done())        next = STATE_CLEANING;
        break;
    }
    current = next;
}
```

> 参考：好家伙VCC 第18章（综合功能整合），教你如何把各个模块串起来

### 第四级：路径规划

#### 弓字形（Bow Pattern）—— 推荐先从它入手

最实用且最容易实现的覆盖式清扫算法：

```
→→→→→→→→→→→
            ↓
←←←←←←←←←←←
↓
→→→→→→→→→→→
```

**逻辑**：

1. 直行清扫直到碰到障碍
2. 右转 90°，前进一个"机身宽度"
3. 再右转 90°，反向直行
4. 重复

> 参考博客：工程级扫地机器人方案（博客园/CSDN），有详细伪代码

#### 栅格地图（进阶）

- 将房间划分为 15×15cm 的栅格
- 编码器 + MPU6050 推算当前位置
- 状态：0 = 未清扫 / 1 = 已清扫 / 2 = 障碍物
- 遍历策略：DFS 优先去未清扫的相邻格子

> 参考：CSDN「基于STM32的智能扫地机器人（含A\\\*算法+DFS）」

```c
// 所有逻辑在固定周期中断中执行，清晰不乱
void HAL_TIM_PeriodElapsedCallback(TIM_HandleTypeDef *htim) {
    if (htim->Instance != TIM2) return;  // TIM2 = 10ms tick

    Sensors_Update();       // 步骤1：读所有传感器、编码器
```

## 调试注意事项

| 问题 | 原因 | 解决方案 | 相关教程 |
|-|-|-|-|
| LED不亮 | GPIO模式/时钟未使能 | 检查RCC时钟、GPIO配置为推挽输出 | 江科大 P5 |
| 走不直 | 电机差异/轮子打滑 | PID闭环控制，先校准编码器系数 | 江科大 P19-P20，好家伙VCC 第7-9章 |
| PWM没输出 | 定时器通道映射错误 | 检查TIMx_CHy和GPIO的AFIO映射 | 江科大 P15-P16 |
| 超声波误判 | 多探头同时发射互相串扰 | **分时触发**，不要同时发射 | 好家伙VCC 第15章 |
| 卡在角落 | 避障逻辑太简单 | 后退后加**随机转向**，别固定转90° | 好家伙VCC 第15章 |
| 电机启动瞬间系统复位 | 电机启动电流大导致电压骤降 | 电机电源端并联 **4700μF 大电容** | — |
| 漏扫严重 | 纯随机碰撞 | 改用弓字形或栅格规划 | 工程级博客 |
| 电池续航差 | 电机效率低/无休眠机制 | 增加待机状态，使用高效驱动芯片 | 江科大 P44-P45（低功耗） |
| MPU6050 角度漂移 | 温漂、积分累积误差 | 定时校准零偏，或结合编码器做融合 | 江科大 P32-P33 |
| 串口乱码 | 波特率不匹配 | 检查STM32和模块双方的波特率 | 江科大 P25-P27 |
