## 六、学习原则

1. **跑通 > 看懂** — 代码能跑比理论精通重要，有 GitHub 证据
2. **一 Phase 不完成不进下个** — 验证标准不打折
3. **硬件驱动 = 面试王牌** — 每个传感器驱动都写 README + 跑通验证
4. **工作日 2h + 周末 4h** — 保持节奏，不搞突击
5. **GitHub 即简历** — 每个 Phase 的代码都有清晰的 commit message + README
